# Showcase Examples

These examples show how the skill turns repositories into stronger public artifacts for jobs, internships, and interviews.

## Example 1: README Repositioning

**Prompt**

`Rewrite this README so recruiters can immediately understand why this project is stronger than a typical CRUD assignment.`

**A strong output should include**

- a sharper one-line positioning statement
- highlights tied to visible repository evidence
- clearer architecture and feature framing
- wording that stays honest about project scope

**Why this showcases the skill**

It demonstrates the core promise of the skill: stronger framing without exaggeration.

## Example 2: Resume Bullet Extraction

**Prompt**

`Read this repository and write 3 interview-defensible resume bullets that reflect the real work.`

**A strong output should include**

- concise bullets with implementation evidence
- technical depth where it exists
- problem-solution framing
- no inflated claims about scale or production usage

**Why this showcases the skill**

This is one of the most concrete and useful public-facing outputs for job seekers.

## Example 3: Portfolio Narrative for an AI Project

**Prompt**

`Package this AI agent project into a portfolio-ready summary, feature list, and interview talking points.`

**A strong output should include**

- a clear narrative arc
- features grouped by what they prove technically
- tradeoffs and constraints discussed honestly
- talking points the owner can actually defend in an interview

**Why this showcases the skill**

It shows that the skill can turn a rough repo into a polished portfolio artifact without drifting into fluff.
